# rhetoric-lens

处理文字的修辞层，方法源自李笑来《写作的真相》第五课「修辞正义」。**第一原则：思想 > 文采。** 文采永远排第二，是锦上添花；内容质量＝逻辑基础框架 + 丰富的细节。修辞的目的是"更好地传递感知"，不是显摆。检测或创作时都先认这条。三种模式：检测（读别人的，扫出全部修辞）、具象（把空话写实）、类比/隐喻（为概念造桥）。

## 核心知识（只用这一套，不要外推）

**四种重要修辞：**

- **具象 = 具体具体再具体**。两条路径：
  - ① 把**概括**的变具体：「我的家乡」→「母校门口拐角那棵柳树，我在树下等过一个女生」。
  - ② 把**抽象**的变具体：用一个可感知的参照物落地数字/程度。例：300 万精子 ≈ 银河系的星星数（对比 400 个卵子 ≈ 太阳系星星数）；中奖概率 < 半夜从床上摔到地板摔死的概率。
- **排比**：本质是大规模并列。用好它的核心不是文笔，而是**研究能力 / 素材量**——你得真有那么多能摆在一起的东西。有素材，直接摆在一起即可（甚至不需要 and/or）。
- **类比 = 架在已知与未知之间的桥梁**。公式：**X ≈ A → 讲 A → X 不言自明**。
  - 经典：教育像眼镜、地球构造像煮熟的鸡蛋、原子内部像太阳系、烂钟一天也能对两次。
  - 三大难点：① **知己知彼**——必须知道读者懂什么、不懂什么（别用比 X 更难懂的 A）；② **跨界**——喻体 A 通常要到 X 所在专业领域之外、日常之中去找；③ **只在某一特定层面相似，而非全等**（全等就不用类比了），并提防读者自己滥用类比。
  - 类比里常顺带用到**借代**（如"透过教育的镜片"，镜片是眼镜的零件不是教育的），借代很难独立存在，最常寄生在类比中。
- **隐喻 = 类比的简化**：只给出 A，不告诉你 X，让你自己想到"A 和 X 是类似的"。讲故事高手必用隐喻（如《绝命毒师》反复出现的粉色泰迪熊）。

**好语言 / 坏语言标准（孙犁，全是逻辑要求而非文采要求）：**
- 好：明确、朴素、简洁、浮雕感、音乐性、与现实生活紧密联系。
- 坏：干燥无味、没有个性、不正确的方言、胡乱的表现、似是而非的丰富、不和现实呼应。

## 模式一：检测（给一篇文章或一本书，找出其中的修辞）

1. 通读文本（整书则逐章/逐节处理，可分批）。
2. 扫描四种修辞，逐条**摘录原文**并标注类型。
3. 拆解每一条的结构：
   - 类比 → 标出 X（本体）、A（喻体）、相似点、跨了哪个界、相似停在哪个层面。
   - 具象 → 标出是"概括→具体"还是"抽象→具体"，用了什么参照物。
   - 排比 → 数清并列了几项，点出背后的素材/研究量。
   - 隐喻 → 标出 A，推测可能的 X。
4. **点评**：精彩在哪 / 有无越界（A 比 X 还难懂、X≈A 几乎全等等问题）。
5. 汇成**修辞收集本**（对应第五课作业：每个精彩类比都是一笔财富，遇到就收集）。

**输出格式：**
```
# 修辞收集本：《围城》第一章

## 类比
1. 原文："……"
   X（本体）= …  A（喻体）= …  相似点 = …
   跨界 = 从[领域]到[领域]   边界 = 仅在[某层面]相似
   点评：…

## 具象
1. 原文："……"  路径：抽象→具体  参照物：…

## 排比 / 隐喻
…

小计：类比 N 条、具象 N 条、排比 N 条、隐喻 N 条
```

## 模式二：具象（把抽象/概括写具体，兼治写作卡壳）

当用户给一句很空的话（"他个子不高""概率很低""我的家乡很美"）：

1. 判断走**哪条路径**：概括→具体，还是抽象→具体。
2. 概括→具体：不断缩小范围，找到一个用户真正"有料可写"的具体落点（家乡→母校门口那棵树）。
3. 抽象→具体：找一个读者能直接感知的参照物，把数字/程度落地。
4. 产出 **3–5 个**具体化候选，供挑选。

**治 writing block 的玩法**：从卡住的稿子里挑一个很概括或很含混的词，以它为开头，放下一切限制天马行空地往具体写半小时，再回看，往往冒出意外思路。需要时直接带用户玩这个游戏。

**输出格式：**
```
原句（抽象/概括）："概率很低"
路径：抽象 → 具体
候选：
1. ……（参照物：…）
2. ……
3. ……
```

## 模式三：类比 / 隐喻（为概念造桥）

当用户要解释一个难懂的概念 X：

1. 先问/确认**目标读者**懂什么（知己知彼，决定 A 的选材）。
2. **跨界**列出 3–5 个候选喻体 A（到 X 的专业领域之外、读者的日常之中找）。
3. 用公式生成完整段落：`X ≈ A` 开头 → 讲 A 的关键性质 → 用"同样的道理/与之类似"过渡 → 讲 X，让其不言自明。
4. **边界检查**：A 是否比 X 更好懂？相似是否只停在某一特定层面（不是全等）？有无滥用风险？
5. 可选：把它简化成**隐喻**（只留 A，不说 X）。

**输出格式：**
```
概念 X：…   目标读者：…
候选喻体 A：① …  ② …  ③ …
推荐 A：…（跨界自[领域]）
类比成段：
  「X 就好像 A……（讲 A）……同样的道理，X……」
边界检查：仅在[某层面]相似 ✅ / A 比 X 更易懂 ✅ / 滥用风险：…
（隐喻版：只说 A，留 X 给读者自己想）
```

## 使用提醒

- 先认清是**检测**（模式一，读别人的）还是**创作**（模式二/三，写自己的）。
- 始终守住"思想 > 文采"：若用户其实是内容/逻辑有问题，先提醒，别用修辞去掩盖。
- 类比可"凑字数"是真的（讲 A 再讲 X，一件事合理地说两遍），但仍以"传递感知"为目的。
- 姊妹技能 `logic-template-lens`（第四课·逻辑轴）可叠用：它看并列/递进/转折骨架，本技能看修辞层。
