# logic-template-lens

把任意文字看成"元素 + 元素之间的逻辑关系",拆出它的逻辑骨架。方法源自李笑来《写作的真相》第四课「逻辑模板」。

**Status:** v0.1.0 — project-scope, manual install.

## What it does

元素之间只有三种关系——并列、递进、转折,没有第四种。本技能据此提供两个方向:

1. **拆解(读别人的范文)** — 切元素 → 标注两两关系 → 画关系链 → 检查是否命中三种"有力模式"(递进+递进 / 递进+转折 / 转折+递进) → 回答"他是怎么做到的"。
2. **诊断(改自己的草稿)** — 同样切元素标关系,对"通篇只有并列 = 平铺直叙"发出预警,并给出可直接套用的升级句式(补"更为重要的是…"造递进、补"竟然/却"造转折)。

讲故事场景额外支持:转折＝反转,标出小人物 / 大目标 / 障碍递进层级 / 反转点。

## Invoke

```
/writing-truth:logic-template-lens <粘贴文本 或 .md/.txt 路径>
```

先说清是"拆别人的"还是"诊断自己的";粒度自适应(短文按句/理由,长文按段/章节)。

## Pairs with

- `rhetoric-lens`(第五课·修辞轴)—— 本技能看逻辑骨架,它看具象/排比/类比/隐喻,可叠用。
- `dissect-author-mind`(第一课·词性感知)—— 更底层的逐词拆解。

## Constraints

只拆解、只诊断,不替用户代笔成文。结论一律落到"他是怎么做到的,我该怎么做到"。
