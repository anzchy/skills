# logic-template-lens

把任意文字看成"元素 + 元素之间的逻辑关系"。方法源自李笑来《写作的真相》第四课「逻辑模板」。读一篇文章，你看到的是成品；这套透镜让你看见骨架——哪些元素被摆在一起、它们之间是并列、递进还是转折。既能拆别人的范文（"他是怎么做到的"），也能诊断自己的草稿（"我这段为什么平淡"）。

## 核心知识（只用这一套，不要外推）

**元素**：一切能"摆在一起"的东西都是元素——可以是词、句、段、理由、陈述、章节。能摆在一起的前提：同一范畴、相同属性。

**元素之间只有三种逻辑关系，没有第四种：**

| 关系 | 本质 | 体现 | 常见信号词 |
|------|------|------|-----------|
| 并列 | and / or | 思考**丰富**（能并列的越多越丰富） | 和、与、也、还、又、或、而且、排比、分号 |
| 递进 | "更" | 思考**深入** | 更、甚至、不仅…而且、更为重要的是、更可怕的是、还没结束 |
| 转折 | "却" | 思考**全面**（找不到转折＝不全面） | 却、但、然而、竟然、居然、没想到、反而、谁知 |

**递进的三条路径**（按价值排序）：从简单到复杂 < 从次要到重要 < **从已知到未知**（最有价值，对应"未知原则"——给读者他不知道的东西）。

**逻辑模板**：3 个元素两两组合共 9 种排列，其中 **3 种"有力模式"**穿透力最强，共性是每种都含递进：
1. 递进 + 递进
2. 递进 + 转折
3. 转折 + 递进

**判废标准**：通篇只有并列 = 平铺直叙 = 拖沓、无聊。递进让它更深入，转折让它更全面。

**讲故事的高级应用**：转折＝反转。好故事 = 小人物 + 大目标 + 层层递进的障碍（≥3 个，递进越来越强）+ 绝境后的反转。开头是英雄、结尾还是英雄而中间无转折 = 傻故事。

## 模式一：拆解（阅读范文）

当用户给一篇文章/段落、想搞懂"他是怎么做到的"：

1. **切元素**：把内容切成有意义的元素（句/段/理由/章节，按粒度选）。
2. **标关系**：相邻元素两两之间标注并列 / 递进 / 转折，画成关系链。
3. **看模式**：是否命中三种有力模式？递进走的是哪条路径（简单→复杂 / 次要→重要 / 已知→未知）？
4. **如果是故事**：标出小人物、大目标、障碍递进层级、反转点。
5. **回答"他是怎么做到的"**：一句话总结这段文字的逻辑设计。

**输出格式：**
```
元素拆分：
① …
② …
③ …

关系链：① —[并列]— ② —[转折]— ③
命中有力模式：转折+递进 ✅（或：无，三元素均为并列）
递进路径：从已知到未知
他是怎么做到的：……
```

## 模式二：诊断（自己的草稿）

当用户给自己写的草稿、想知道好不好：

1. 按模式一切元素、标关系。
2. **平铺直叙预警**：若相邻关系全是并列，明确指出"此段只有并列 → 平淡"。
3. **升级建议**（给可直接套用的句式）：
   - 造递进：在某个理由后补一句"更为重要的是…"/"更可怕的是…"，再补一个比前面更重要/更惊人/更少人知道的元素。
   - 造转折：在某处插入"却 / 竟然 / 没想到"，补一个反方向的元素。
4. **丰富度反问**：举例是否只举了一个？理由是否只给了一条？感知是否只有单一感官？（"会写"的真相是"有东西可写"，靠素材积累不靠聪明。）

**输出格式：**
```
当前骨架：① —[并列]— ② —[并列]— ③
诊断：⚠️ 全并列，平铺直叙
升级方案：
- 把 ②③ 改成递进：在②后加"更为重要的是"，③替换为一个更…的元素
- 或在③前加转折"竟然"，补一个反方向例子
丰富度检查：第②点只举了一个例子，建议再并列 2 个
```

## 使用提醒

- 先问清是**拆别人的**（模式一）还是**诊断自己的**（模式二）。
- 粒度自适应：短文按句/理由切，长文/整书按段/章节切。
- 不要堆砌术语，目标是让用户看完能复用——每次都落到"他是怎么做到的，我该怎么做到"。
- 姊妹技能 `rhetoric-lens`（第五课·修辞轴）可叠用：本技能看逻辑骨架，它看具象/排比/类比/隐喻。
