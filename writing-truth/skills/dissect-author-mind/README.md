# dissect-author-mind

把一篇文章按词性拆开,"钻进作者的脑子"。方法源自李笑来《写作的真相》第一课「如何钻进作者的脑子」。

**Status:** v0.1.0 — project-scope, manual install.

## What it does

给定一段中文文本,逐词标注词性并反推作者的认知与感知:

1. **圈名词** — 区分具体名词 / 抽象名词,算抽象名词占比(知识分子文章常见 15%–25%)。
2. **划形容词** — 看名词前的修饰,判断作者的感知能力强弱。
3. **标动词、标副词** — 同理拆解动作与对动作的感知,提示冗余副词。
4. **生成彩色标注 HTML** — 五色区分词性(具体名词蓝 / 抽象名词深蓝加粗 / 形容词绿下划线 / 动词红 / 副词橙虚线),配数据统计表与文字解读,每条结论附文中原词为证。
5. **反推画像** — 受教育程度、感知能力、动作感知,各有判据。

## Render path

> 依赖:首选渲染路径需要同 marketplace 的 `anzchy-skills` 插件中的 `jack-html-preview`;两个插件都安装时体验最佳,缺失则自动回退到直接生成 HTML。

## Constraints

只标注、只分析,不改写、不润色、不评判文章好坏。文本 >3000 字会先提示分段。
