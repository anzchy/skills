# dissect-author-mind

读一篇文章时,你看到的只是结果;李笑来在《写作的真相》第一课教的"神技"是:把名词圈出来、把形容词划出来,你就"钻进了作者的脑子",看到他眼睛看到的世界。本技能把这套方法固化下来——对输入文本逐词标注词性,生成彩色标注 HTML,再用词性分布反推作者的认知与感知。这不是语法练习,而是一台"感知透视镜"。

待拆解的文本:$ARGUMENTS

## 起始状态
用户提供一段中文文本(说理文/小说/散文/纪实文皆可),来源可以是粘贴的文字、`.md`/`.txt` 文件路径,或剪贴的片段。

## 目标产物
一个 `<原名>-拆解.html`,用浏览器打开即见彩色标注全文 + 数据统计 + 文字解读。**生成方式见"渲染 HTML"一节:优先调用 `/anzchy-skills:jack-html-preview`,不存在则直接写出自包含 HTML。**

## 执行步骤(严格按序,逐句处理,不跳词)

1. **圈名词(蓝色)** — 标出所有名词,并区分:
   - 具体名词(能看见摸到的:桌子、网红)→ 浅蓝 `#2563eb`
   - 抽象名词(看不见的概念:落后、自由、竞争力)→ 深蓝加粗 `#1e3a8a` bold
2. **划形容词(绿色 `#16a34a`,下划线)** — 重点标注名词前的形容词;若一个名词前排列多个形容词,全部保留,它们暴露作者对同一事物的多重(甚至对立)感知。
3. **标动词(红色 `#dc2626`)** — 标出充当谓语的动作词,区分具体动作与抽象动作。
4. **标副词(橙色 `#ea580c`,虚线下划线)** — 标出修饰动词/形容词的副词;记录副词是否准确、是否冗余。

## 数据统计(自动计算并展示)
- 各词性数量与占全文实词比例
- **抽象名词占比**(关键指标):标注落点 → <5% 偏低 / 10%–25% 知识分子常见区间 / >30% 异常
- 名词前平均形容词数、动词前平均副词数

## 文字解读(基于李笑来判据,每条须给出文中证据)
- **受教育程度**:由抽象名词占比与质量推断——能正确使用清晰、准确、必要的抽象概念越多,受教育程度越高;全为具体名词则偏低。
- **感知能力**:由形容词推断——名词前无形容词=感知弱;有且精准=感知强;有且精炼必要=感知与表达俱强;同一名词前多个对立形容词=感知丰富且精巧。
- **动作感知**:由动词/副词关系同理推断,并提示是否存在可删的冗余副词。

## 渲染 HTML(关键:先查 skill,再决定路径)

1. **优先调用 `/anzchy-skills:jack-html-preview`**:若该 skill 可用,先把"彩色标注全文 + 统计 + 解读"整理成一个临时 Markdown 文件(`<原名>-拆解-source.md`,内含上色用的 inline HTML `<span>` 标签),再读取其 `reference/claude-design-spec.md`,按 Claude 设计语言渲染成自包含 HTML;把三个交互特性落到本场景:① 可折叠章节树 ② 点击图例按词性高亮(其余词淡出) ③ 方法流程图(圈名词→划形容词→标动词→标副词→反推感知)。
2. **不存在则直接生成**:若 `/anzchy-skills:jack-html-preview` 不可用,直接写出 `<原名>-拆解.html`——单文件、所有 CSS 内联、不依赖网络、双击即开。结构为:① 顶部配色图例 ② 彩色全文 ③ 数据统计表 ④ 文字解读。

## 约束
- 只标注、只分析,**不改写、不润色、不评判文章好坏**。
- 词性有歧义时,按其在句中实际成分判定(做主语/宾语→名词,做谓语→动词),不确定的词留原色并在解读区列出。
- 仅生成所需文件,不创建多余文件、不引入外部依赖,样式用内联 CSS。
- 文本超长(>3000 字)时,先告知将分段处理并确认,再继续。

## 完成判据
HTML 已生成且每个实词都已着色;统计含抽象名词占比;解读区每条结论都附文中原词为证。完成后输出:✅ 已生成 <文件路径>,抽象名词占比 X%(渲染方式:jack-html-preview / 直接生成)。
