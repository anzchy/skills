# writing-truth — plugin guide

阅读/写作技能集，源自李笑来《写作的真相》。三个 skill 按"词性 → 逻辑 → 修辞"三层组织。

## 何时用哪个

- 想看**作者的感知/受教育程度**、做词性拆解 → `dissect-author-mind`（第一课）。
- 想看/改**逻辑骨架**（这段是不是平铺直叙、元素怎么排列）→ `logic-template-lens`（第四课）。
- 想**检测一篇文章或一本书里的修辞**、把空话写具体、为概念造类比 → `rhetoric-lens`（第五课）。

## 原则（贯穿三者）

- **思想 > 文采**：修辞只为更好传递感知，不为显摆。若是内容/逻辑有问题，先提醒，别用修辞掩盖。
- **只拆解、只诊断，不代笔成文**；结论落到"他是怎么做到的，我该怎么做到"。
- 三者可叠用：同一段文字先 dissect 看词性，再 logic 看关系链，最后 rhetoric 看修辞。

## 知识边界

每个 SKILL.md 的"核心知识"严格取自对应课的原文，不要外推到书里没有的框架。`knowledge/` 目录可放三课原文摘录作为共享底座。
